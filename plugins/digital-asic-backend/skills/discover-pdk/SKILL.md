# Discover PDK

Scan a PDK directory, identify all files required by the backend flow, confirm selections with the user, and write a `pdk.md` summary file to the PDK directory for use by `/new-project`.

**Argument**: `$ARGUMENTS` = path to PDK root directory.

If not provided, ask the user for the PDK path.

## Step 1: Check for Existing pdk.md

If `$PDK_PATH/pdk.md` already exists:
- Read and display its contents to the user.
- Ask: **"A PDK summary already exists for this directory. Use it as-is, or re-discover?"**
- If the user chooses to use it: report success and exit.
- If the user chooses to re-discover: continue with Step 2.

## Step 2: Search for PDK Files

Run the following searches. Collect all candidate paths for each file type.

```bash
PDK_PATH="$ARGUMENTS"

# Standard cell NDM (exclude phy_only and antenna variants)
find $PDK_PATH -name "*.ndm" -not -path "*phy_only*" -not -path "*antenna*" | sort

# Physical-only NDM
find $PDK_PATH -name "*.ndm" -path "*phy_only*" | sort

# Tech file
find $PDK_PATH -name "*.tf" | sort

# TLU+ parasitic files
find $PDK_PATH \( -name "*.TLUplus" -o -name "*.tluplus" \) | sort

# TLU+ layer map file
find $PDK_PATH -name "*.itf" -o -name "*.map" | grep -i "tlu\|itf\|layer" | sort

# Timing DB files (look for tt, ss, ff corners)
find $PDK_PATH -name "*.db" | sort

# Symbol DB files
find $PDK_PATH -name "*.sdb" | sort

# StarRC TCAD grid files (worst and best corners)
find $PDK_PATH -name "*.nxtgrd" | sort

# Antenna rules TCL
find $PDK_PATH -name "*.tcl" | xargs grep -l "define_antenna_rule\|define_antenna_layer_rule" 2>/dev/null | sort
```

## Step 3: Present Candidates and Confirm

Present a table of the best candidate for each required variable. If multiple candidates exist for a slot, list all options and ask the user to choose.

```
=== PDK Discovery: $PDK_PATH ===

| Variable              | Selected Path                          |
|-----------------------|----------------------------------------|
| NDM_REFERENCE_LIB     | ...                                    |
| NDM_REFERENCE_LIB_PHY_ONLY | ...                               |
| TECH_FILE             | ...                                    |
| MAX_TLUPLUS_FILE      | ...  (worst-case corner)               |
| TYP_TLUPLUS_FILE      | ...  (typical corner)                  |
| MIN_TLUPLUS_FILE      | ...  (best-case corner)                |
| TLUPLUS_MAP_FILE      | ...                                    |
| DB_TT_FULL_PATH       | ...  (TT corner timing DB)             |
| DB_SS_FULL_PATH       | ...  (SS corner timing DB)             |
| TARGET_LIBRARY        | ...  (basename of TT .db)              |
| DB_DIR                | ...  (directory of .db files)          |
| SYMBOL_LIBRARY        | ...  (basename of .sdb)                |
| SDB_DIR               | ...  (directory of .sdb files)         |
| TCAD_GRD_WST          | ...  (worst nxtgrd)                    |
| TCAD_GRD_BST          | ...  (best nxtgrd)                     |
| ANTENNA_RULES_TCL     | ...                                    |
```

**Heuristics for corner identification:**
- TT corner DB: filename contains `tt` (case-insensitive)
- SS corner DB: filename contains `ss`
- Max TLU+: filename contains `max`, `worst`, or `rcworst`
- Typ TLU+: filename contains `typ`, `typical`, or `rctypical`
- Min TLU+: filename contains `min`, `best`, or `rcbest`
- Worst nxtgrd: filename contains `wst`, `worst`, or `max`
- Best nxtgrd: filename contains `bst`, `best`, or `min`

Ask the user to confirm or correct the selection before writing.

If any required file is **not found**, flag it clearly:
> **Missing: ANTENNA_RULES_TCL** — no TCL file with antenna rule definitions found in $PDK_PATH. You must provide this path manually before using `pdk.md` with `/new-project`.

## Step 4: Write pdk.md

Write `$PDK_PATH/pdk.md` with the confirmed paths. Use this exact format so `/new-project` can parse it reliably:

```markdown
# PDK Summary: <basename of PDK path>

<!-- Generated by /discover-pdk on <date>. Edit manually to correct paths. -->

## Library Configuration

| Variable | Value |
|----------|-------|
| LIBRARY_PATH | /path/to/pdk |
| DB_DIR | /path/to/db/dir |
| SDB_DIR | /path/to/sdb/dir |
| TARGET_LIBRARY | tt_corner.db |
| SYMBOL_LIBRARY | stdcell.sdb |

## Physical Libraries

| Variable | Value |
|----------|-------|
| NDM_REFERENCE_LIB | /full/path/to/stdcell.ndm |
| NDM_REFERENCE_LIB_PHY_ONLY | /full/path/to/stdcell_phy_only.ndm |
| TECH_FILE | /full/path/to/tech.tf |

## TLU+ Parasitics

| Variable | Value |
|----------|-------|
| MAX_TLUPLUS_FILE | /full/path/to/max.TLUplus |
| TYP_TLUPLUS_FILE | /full/path/to/typ.TLUplus |
| MIN_TLUPLUS_FILE | /full/path/to/min.TLUplus |
| TLUPLUS_MAP_FILE | /full/path/to/map.itf |

## StarRC

| Variable | Value |
|----------|-------|
| TCAD_GRD_WST | /full/path/to/worst.nxtgrd |
| TCAD_GRD_BST | /full/path/to/best.nxtgrd |

## Timing Corners

| Variable | Value |
|----------|-------|
| DB_TT_FULL_PATH | /full/path/to/tt_corner.db |
| DB_SS_FULL_PATH | /full/path/to/ss_corner.db |

## Constraints

| Variable | Value |
|----------|-------|
| ANTENNA_RULES_TCL | /full/path/to/antenna_rules.tcl |
```

If a variable has no value (file not found and not provided by user), write `MISSING` as the value. `/new-project` will warn on any `MISSING` entries.

## Step 5: Confirm to User

Report the path to the written file and next steps:

```
PDK summary written to: $PDK_PATH/pdk.md

Next step: run /new-project <pdk-path> <project-path> <design-name>
```
