# v1.2.0 Release Notes

## Summary

| Area | Change |
|------|--------|
| `fc/scripts/stages/` | Each PnR stage copies the upstream checkpoint into its own block before editing — upstream checkpoints stay pristine across runs |
| `fc/scripts/stages/floorplan.tcl` | Std-cell rail strategy extended to `outermost_ring` (matches mesh strategy) |
| `fc/scripts/stages/floorplan.tcl` | Removed orphan `compile_pg -strategies std_cell_strategy` (referenced strategy before it was defined), duplicate `set_ignored_layers`, and `pg_mesh_via_rule` (mesh now uses default via rule) |
| Repo docs | Added `CLAUDE.md` at marketplace root and per-plugin; `releases/` is now the single source of release notes (replaces the old `CHANGELOG.md`) |

The functional changes are all template-only. Existing projects keep their copy of `fc/scripts/` until re-scaffolded — see migration notes below to port manually.

## Stage Checkpoint Isolation

### Problem

Fusion Compiler edits the open block in memory. The previous flow opened the upstream checkpoint, modified it, and called `save_block -as ${DESIGN_NAME}_<this>`. Because `save_lib` then committed the entire dirty library to disk, the upstream block on disk was **also** mutated. Re-running a stage no longer started from a clean source.

For example, before v1.2.0, running `synthesis` on `_floorplan` would mutate `_floorplan` itself in addition to producing `_synthesis`.

### Fix

Each stage now opens its required checkpoint as a **read-only source**, copies it into a fresh block named after the produced checkpoint, opens that copy, and operates on it:

```tcl
######################################################################
# Open Design — copy from <prev> so the source checkpoint stays clean
######################################################################
if {[sizeof_collection [get_blocks -quiet ${DESIGN_NAME}_<this>]] > 0} {
    remove_block -force [get_blocks ${DESIGN_NAME}_<this>]
}
copy_block -from ${DESIGN_NAME}_<prev> -to ${DESIGN_NAME}_<this>
open_block ${DESIGN_NAME}_<this>
```

At the end of the stage, plain `save_block` (no `-as`) writes back to the already-correctly-named block.

Re-running a stage discards its previous output block and re-copies from the upstream checkpoint, so each run starts from a clean source and the upstream block is never mutated.

### Stages affected

`floorplan`, `synthesis`, `placement`, `clocktree`, `routing`, `dfm`. Three are intentionally unchanged:

- `init` — no upstream checkpoint to preserve.
- `output` — opens `_dfm` to write outputs but never calls `save_block`, so the disk copy stays clean.
- `eco` — intentionally edits `_dfm` in place across iterations; safety is provided by the `.dlib` backup that `/run-eco` already takes.

The contract documented in `/run-fc-step` (which checkpoint each step requires/produces) is unchanged.

## Floorplan PG Cleanup

### Std-cell rail extends to outermost ring

`set_pg_strategy std_cell_strategy` now declares `-extension {stop: outermost_ring}`, matching `pg_mesh_strategy`. Previously the rail stopped at the core boundary, leaving a gap between the rail termination and the PG ring.

### Orphan and duplicate code removed

Two leftover lines from a prior reorganization:

- `compile_pg -strategies std_cell_strategy -tag pg_rail` was being called before `std_cell_strategy` was defined (an undefined-strategy reference).
- `set_ignored_layers -min_routing_layer METAL1 -max_routing_layer METAL6` was duplicated — once between mesh and rail, once after rail. Only the post-rail call is kept.

### `pg_mesh_via_rule` removed

The 5-clause `set_pg_strategy_via_rule pg_mesh_via_rule { ... }` block and the `-via_rule {pg_mesh_via_rule}` argument on `compile_pg` are removed. The mesh now compiles with FC's default via rule, which auto-creates vias at strategy/ring intersections. **Generated vias may differ slightly** — re-extract and re-time existing layouts after porting.

The result: PG section reads top-down as ring → mesh → rail, each item defined immediately before it is compiled, no orphan references.

## Migrating an Existing Project

The functional changes live in `fc/scripts/`, which is copied per project at `/new-project` time and not re-synced afterward. To port v1.1.1 projects:

```bash
cd $PROJECT_PATH/fc/scripts/stages

# 1. For each of: floorplan synthesis placement clocktree routing dfm
#    Replace the "open_block" prologue with the copy-and-open block, and
#    drop the "-as <name>" from save_block. Pattern:
#
#      OLD:  open_block ${DESIGN_NAME}_<prev>
#            ...
#            save_block -as ${DESIGN_NAME}_<this>
#
#      NEW:  if {[sizeof_collection [get_blocks -quiet ${DESIGN_NAME}_<this>]] > 0} {
#                remove_block -force [get_blocks ${DESIGN_NAME}_<this>]
#            }
#            copy_block -from ${DESIGN_NAME}_<prev> -to ${DESIGN_NAME}_<this>
#            open_block ${DESIGN_NAME}_<this>
#            ...
#            save_block

# 2. In floorplan.tcl, in the std_cell_strategy definition, add:
#       -extension {stop: outermost_ring}
#
#    And remove these orphans (between the mesh and rail blocks):
#       set_ignored_layers -min_routing_layer METAL1 -max_routing_layer METAL6
#       compile_pg -strategies std_cell_strategy -tag pg_rail
#
#    And remove the pg_mesh_via_rule definition + its -via_rule arg on compile_pg.
```

Or simpler: re-scaffold a fresh project with `/new-project` and copy your `hdl/`, `constraints/timing.tcl`, and floorplan/io constraints over.
